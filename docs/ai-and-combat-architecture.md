# AI and Combat Architecture

현재 결론: 다음 기능은 `명령`, `이동`, `전투`, `무기 데이터`를 분리한 현재 흐름을 유지하면서 작은 모듈 단위로 추가한다.

## Purpose

이 문서는 전차/보병/무기 시스템이 어디에 속하는지 정리해서, 기능을 추가할 때 한 파일에 코드가 몰리거나 AI 판단이 다시 꼬이는 것을 막는다.

## Current System Flow

1. `Game`은 입력, 지휘관, 유닛 업데이트, 전투 업데이트, 점령지 업데이트, 렌더링을 순서대로 실행한다.
2. `CommanderAI`는 팀별 전차/보병에게 거점 목표를 배정한다.
3. `CommanderAI`는 같은 거점을 바라보는 전차와 분대를 `CombinedArmsOrder`로 묶고, 전차에는 엄호 위치를, 분대에는 진입 명령을 준다.
4. `SquadAI`는 보병을 분대 단위로 묶고, 분대 안에서 돌격/지원/경계 역할을 나눠준다.
5. `TankAI`는 명령 목표를 받아 이동은 `NavigationAgent`, 교전은 `CombatController`로 나눠 처리한다.
6. `InfantryAI`는 분대 역할을 반영해서 목표 이동, 적 탐지, 사격, 제압, 엄폐를 처리한다.
7. `CoverSlotManager`는 엄폐 지점 예약을 관리해서 여러 보병이 같은 엄폐점에 겹치는 현상을 줄인다.
8. `combat.js`는 전차 포탄, 보병 총격, 폭발, 연막, 제압 효과를 처리한다.
9. `renderer.js`와 `hud.js`는 현재 상태를 보여주고, 디버그 오버레이는 AI 판단 근거를 보여준다.

## Key Data

- `src/data/constants.js`: 팀, 전차 탄약, 전차 AI 공통 수치.
- `src/data/infantry-weapons.js`: 보병 무기 데이터. 사거리, 피해, 연사, 명중 보정, 제압량, 시각 길이를 여기에서 조정한다.
- `src/data/map01.js`: 맵, 스폰, 거점, 내비게이션 그래프, 보병 무장 배치.
- `world.safeZones`: 스폰 준비 구역. 현재는 아군 시작 지점에 있으며, 플레이어 도보 상태의 무기 선택과 초반 보호를 담당한다.
- `Game.squads`: 현재 보병 분대 목록. 팀별 보병 스폰 후 `SquadAI`가 생성된다.
- `CommanderAI.operations`: 현재 진행 중인 전차-보병 협동 작전 목록. 각 작전은 거점, 엄호 전차, 진입 분대, `supportPoint`를 가진다.
- `Order.operationId`: 협동 작전 ID. 같은 ID를 받은 전차와 분대는 같은 거점을 두고 역할을 나눠 움직인다.
- `Order.supportPoint`: 협동 작전에서 전차가 점령원 안으로 들어가지 않고 엄호할 위치.
- `InfantryUnit.squadId` / `InfantryUnit.squadRole`: 보병이 속한 분대와 역할. 현재 역할은 `assault`, `support`, `security`다.
- `CoverSlotManager.reservations`: 엄폐 위치 예약 테이블. 예약은 짧은 시간마다 갱신되고, 유닛이 죽거나 시간이 지나면 해제된다.
- `coverQuality` / `coverMetrics`: 보병이 엄폐 후보를 고를 때 쓰는 점수. 위험 차단, 사격각, 목표 유지, 이동 비용, 위협과 거리로 계산한다.
- `InfantryUnit.weaponId`: 보병이 들고 있는 무기 ID.
- `player.weaponInventory`: 플레이어 도보 상태에서 1/2/3으로 바꿀 수 있는 무기 목록.
- `InfantryUnit.suppression`: 총격과 폭발로 쌓이는 제압 수치. 높으면 이동/사격/점령 효율이 떨어진다.

## Current Map Layout

- 맵 크기는 `3400 x 2200`이며, 좁은 3거점 전선이 아니라 A/B/C/D 4거점으로 나뉜다.
- `CommanderAI`는 아군 `A -> B -> D -> C`, 적군 `C -> D -> B -> A` 순서로 압박한다.
- 전차는 병력이 3대 이상이면 최대 3개 거점에 나뉘어 배정되고, 보병은 5명 이상이면 최대 3개 거점으로 분산된다.
- 보병은 현재 팀당 1개 분대로 묶이며, 분대 안에서 기관총은 지원, 기관단총과 선두 보병은 돌격, 나머지는 경계 역할을 받는다.
- `src/data/map01.js`의 `navGraph.objectiveNodes`는 모든 거점이 반드시 내비게이션 그래프에 연결되도록 유지해야 한다.

## Current Bug Watchlist

- 보병 엄폐는 v1 품질 점수를 쓴다. 위험 차단/사격각/목표 유지/이동 비용을 보지만, 분대별 엄폐 우선권과 전차 사격선 회피는 아직 없다.
- 보병 사격은 즉발 판정이다. 탄환이 실제 물체로 날아가는 구조가 아니라서, 나중에 탄속/투사체 회피가 필요하면 별도 시스템이 필요하다.
- 전차-보병 협동 명령은 v1이다. 전차는 엄호 위치를 받고 보병은 진입하지만, 전차 사격선에 보병이 들어가지 않게 조정하는 세부 판정은 아직 없다.
- 사망한 보병은 시각적으로 남지만, 시간이 지나면 제거되는 정리 시스템은 아직 없다.
- 점령지 AI는 목표 우선순위가 단순하다. 적 저항이 강한 거점을 회피하거나 우회하는 전략 판단은 아직 없다.

## Recommended Next Order

1. 기본 버그 안정화: 플레이어 사격, 보병 이동 떨림, 스폰 안전구역처럼 체감 기반이 되는 문제를 먼저 고친다.
2. `SquadAI` 확장: 현재 돌격/지원/경계 역할에서 사격조/이동조 교대와 분대 후퇴 판단을 추가한다.
3. 엄폐 품질 튜닝: 현재 점수 기준을 실제 플레이에서 보며 너무 소극적/공격적인 값만 조정한다.
4. 협동 명령 튜닝: 엄호 전차가 너무 멀거나 너무 가까운지, 보병 진입 타이밍이 자연스러운지 플레이로 확인한다.
5. 전차 사격선 회피: 보병이 아군 전차 포구 앞을 지나가지 않도록 위험선을 추가한다.
6. 무기 확장: 산탄총, 저격총, 대전차 로켓 같은 특수 무기는 `infantry-weapons.js`에 추가하고, 필요할 때만 전용 판정을 붙인다.
7. 사망/잔해 정리 시스템: 전사자 표시 유지 시간과 제거 규칙을 만든다.

## Next Large AI Axis

현재 큰 구조는 `CombinedArmsOrder`를 두고 `CommanderAI -> SquadAI/TankAI`로 명령을 나누는 방식이다.

- `CommanderAI`: 어느 거점을 공격/방어할지 정하고, 전차와 보병을 같은 작전 묶음으로 배정한다. 현재는 분대 하나에 엄호 전차 하나를 우선 연결한다.
- `SquadAI`: 보병 여러 명을 한 분대로 묶고, 일부는 사격, 일부는 이동, 일부는 엄폐를 맡긴다. 현재 최소 구현은 돌격/지원/경계 역할 배정까지다.
- `TankAI`: 기존 이동/전투 구조를 유지하되, 협동 명령을 받으면 거점 안으로 무리하게 들어가기보다 `supportPoint`에서 보병 접근로를 엄호한다.
- `CombatController`: 적을 쫓는 판단보다 “내 명령 구역을 위협하는 적”을 우선한다.
- `NavigationAgent`: 분대와 전차가 같은 좁은 지점에 몰리지 않도록 서로 다른 접근 슬롯을 받는다.

## Add Feature Rule

새 기능을 넣을 때는 먼저 이 질문을 통과한다.

- 데이터만 추가하면 되는가? 그러면 `src/data/*`에 둔다.
- 판단이 바뀌는가? 그러면 `src/ai/*`에 둔다.
- 피해/탄도/제압이 바뀌는가? 그러면 `src/systems/combat.js` 또는 새 전투 시스템으로 뺀다.
- 화면 표시만 바뀌는가? 그러면 `renderer.js`나 `hud.js`에 둔다.
- 두 군데 이상에 걸치면, 먼저 작은 데이터/상태 필드를 만들고 AI와 렌더링을 나중에 연결한다.
