# AI and Combat Architecture

현재 결론: 전차, 보병, 병과, 장비, 명령, 이동, 전투는 서로 분리된 작은 시스템으로 유지한다.

## Purpose

이 문서는 2D 탱크 프로토타입에서 AI와 전투 시스템이 어디에 나뉘어 있는지 정리한다. 새 기능을 붙일 때 한 파일에 코드가 몰리지 않게 하는 기준이다.

## Current System Flow

1. `Game`은 배치 화면, 시작 카운트다운, 지휘관, 분대, 보병, 전차, 투사체, 거점, 렌더링 순서로 업데이트한다.
2. `CommanderAI`는 팀별 전차와 보병 분대에 거점 명령을 배정한다.
3. `CommanderAI.operations`는 전차-보병 협동 작전을 기록한다. 전차는 `supportPoint`로 가서 엄호하고, 보병 분대는 거점으로 진입한다.
4. `SquadAI`는 일반 보병을 `assault`, `support`, `security` 역할로 나눠 진형 위치를 만든다. 정찰병은 분대 진입에서 빠져 별도 정찰 명령을 받는다.
5. `TankAI`는 이동을 `NavigationAgent`, 교전을 `CombatController`에 맡긴다.
6. `InfantryAI`는 목표 이동, 엄폐, 제압, 사격을 처리한다.
7. `CoverSlotManager`는 보병 엄폐 자리가 겹치지 않게 예약한다.
8. `combat.js`는 전차 포탄, 보병 총격, 수류탄, RPG, 수리킷 효과를 처리한다.
9. `physics.js`는 벽 충돌, 전차 간 간격, 보병-전차 겹침 방지를 처리한다.

## Key Data

- `src/data/constants.js`: 팀, 탄약, 전차 AI 공통 수치.
- `src/data/infantry-weapons.js`: 보병 장비 데이터. 총기, 수류탄, RPG, 수리킷의 사거리, 쿨다운, 피해, 시각 크기를 정의한다.
- `src/data/infantry-classes.js`: 플레이어 병과 데이터. `보병`, `공병`, `정찰`과 각 병과의 1/2/3 장비 슬롯을 정의한다.
- `player.classId`: 현재 플레이어 병과.
- `player.weaponInventory`: 현재 병과의 장비 슬롯 목록.
- `player.activeSlot`: 현재 선택된 장비 슬롯.
- `player.equipmentAmmo`: 수류탄, RPG, 수리킷 같은 제한 장비의 잔량.
- `world.safeZones`: 플레이어가 병과를 선택하고 재보급하는 안전구역.
- `world.reconPoints`: 정찰병이 분대와 떨어져 이동할 외곽 감시 지점.
- `CommanderAI.operations`: 현재 전차-보병 협동 작전 목록.
- `Order.supportPoint`: 엄호 전차가 점령지 안으로 들어가지 않고 자리 잡는 위치.
- `game.teamReports`: 정찰병이 보고한 적 위치. 전차 AI가 직접 시야 밖 목표를 추적하고 사선 확보를 시도할 때 참고한다.
- `game.deploymentOpen`: 시작 전 병과 선택/작전 지도 화면이 열려 있는 상태.
- `game.countdownStarted`: 배치 화면을 닫고 실제 출격 카운트다운이 시작된 상태.
- `game.matchTime`: 출격 이후 Tab 점수판에 표시되는 경기 시간.

## Current Map And Deployment Rules

- `map01`은 기존 전장을 약 1.32배 확장한 뒤 추가 기지 공간을 붙여 약 `5468 x 3664` 크기로 사용한다.
- 양 팀 기지에는 벽과 게이트가 있으며, 전차는 기지 안 주차 위치에서 시작한다.
- 기지를 구성할 때 안전구역 주변으로 들어온 기존 필드 `building` 장애물은 제거한다. 기지 안 건물 침입과 보병 출구 고립을 막기 위한 규칙이다.
- 플레이어는 블루 기지 안 도보 상태로 시작한다. 처음에는 큰 배치 화면이 열리고, 작전 지도와 `보병`, `공병`, `정찰` 병과 선택 버튼을 보여준다.
- `출격 시작`을 누르면 배치 화면이 닫히고 5초 카운트다운이 시작된다. 이 시간 동안 플레이어는 기지 안에서 움직이고 `E`로 주차된 플레이어 전차에 탑승할 수 있다.
- `Game.matchStarted`가 `false`인 동안 승무원은 탱크에 탑승하지만, AI 보병/전차 명령, 거점 점령, 승패 판정은 정지한다.
- `startCountdown`이 끝나면 `matchStarted`가 `true`가 되고 양 팀 AI가 기지 밖으로 출격한다.
- 기본 템포는 낮아졌다. 전차 기본 속도, 보병 속도, 승무원 속도, 점령 속도를 낮춰 거점 사이 이동과 교전 준비 시간이 길어졌다.
- 양 팀 AI 보병은 14명씩으로 늘었고, 공병과 정찰병은 각 팀 2명씩 배치된다.

## Current Player Class Rules

- 시작 전 배치 화면에서 병과 버튼을 눌러 플레이어 병과를 고른다.
- 카운트다운 중 안전구역 안에서 도보 상태일 때 `1/2/3`은 병과 선택 보조 입력이다.
- 경기 시작 후 도보 상태일 때 `1/2/3`은 현재 병과의 장비 슬롯 선택이다.
- 보병: `1 기관총`, `2 권총`, `3 수류탄`.
- 공병: `1 기관총`, `2 RPG`, `3 수리킷`.
- 정찰: `1 저격총`, `2 권총`.
- 병과를 안전구역에서 다시 선택하면 해당 병과의 제한 장비가 재보급된다.
- 플레이어가 정찰 병과에서 저격총을 들고 `우클릭`을 유지하면 정지 조준 모드가 된다. 카메라는 더 넓게 보고, 조준 오버레이가 뜨며, 저격총 사거리와 명중 보정이 증가한다.

## Current HUD Rules

- 상단 HUD는 FPS 거점 HUD처럼 빈 거점 모양과 점령 색상 변화만 우선 보여준다.
- 이전의 상태/체력/작전 텍스트 패널은 기본 HUD에서 숨긴다.
- 하단 탄약 HUD는 플레이어가 전차에 탑승했을 때만 표시한다. 도보 상태에서는 병과 선택/무기 패널을 띄우지 않는다.
- `Tab`을 누르는 동안 점수판을 표시한다. 현재 점수판은 팀별 전차 수, 보병 수, 전투 가능 유닛 수, 팀 K/D를 보여준다.
- 현재 K/D는 개인 킬 로그가 아니라 팀 손실 기반이다. 상대 팀 손실 수를 팀 킬로 계산한다.

## Current Tank Control Rules

- `E`: 탑승, 하차, 상호작용.
- `좌클릭`: 탑승 중 주포 발사. AP는 즉시 현재 포신 방향으로 나간다. HE가 장전된 상태에서는 클릭한 지점을 `fireOrder`로 잠그고, 포탑이 그 지점에 정렬되면 자동 발사한다.
- `우클릭 유지`: 탱크 조준 모드. 카메라가 살짝 줌아웃되고 조준 방향으로 앞을 더 보여준다. 마우스가 사거리 밖에 있어도 조준점은 보이고, 실제 탄착 가능 지점은 별도 표시된다.
- 마우스: 포탑 목표 방향 지정. 포탑은 즉시 돌아가지 않고 `turretTurnRate`에 맞춰 느리게 따라간다.
- `1`: 철갑탄 장전.
- `2`: 고폭탄 장전.
- `3`: 연막탄 즉시 전개.
- 조준 오차가 클 때 AP를 발사하면 `Tank.fire()`에서 산포가 조금 커져 급하게 쏜 느낌이 난다.
- HE는 클릭한 탄착 지점까지의 거리 신관을 `Tank.fire({ fuseDistance })`로 넘긴다. 벽이나 사거리 제한이 있으면 가능한 지점으로 보정되고, 해당 지점 근처에서 폭발한다.
- 렌더러는 탑승 중 현재 포신 기준 조준선, 마우스 조준점, 실제 탄착 가능 지점, HE 사격 예약 지점, 고폭탄 예상 폭발 범위를 표시한다. 우클릭 조준 중에는 약한 조준 오버레이도 표시한다. 카메라 줌 상태에서는 마우스 월드 좌표와 미니맵 시야 박스도 줌에 맞춰 보정한다.

## Current Ammo Roles

- 철갑탄(AP): 빠르고 직격 피해가 크다. 전차를 직접 맞히는 주력 탄이며 폭발 범위는 거의 없다.
- 고폭탄(HE): AP보다 느리고 폭발 범위가 크다. 플레이어는 원하는 탄착점을 찍어 제압 지점을 만들 수 있다. 보병 피해와 제압은 강하지만 전차 피해는 `tankDamageScale`로 낮게 들어간다.
- 전차 AI는 적 전차에는 AP를 우선하고, 보병/승무원/보병 밀집 지점에는 HE를 우선한다. HE 발사 시 목표까지의 거리 신관을 사용해 목표 근처에서 폭발하게 한다.
- 전차 AI는 HE 폭발 반경 가까이에 아군 보병, 승무원, 도보 플레이어가 있으면 HE 대신 AP를 고르거나 발사를 보류한다.
- 전차 AI는 목표가 바뀌었을 때 아직 장전 초반이면 더 적절한 탄으로 재장전을 시작할 수 있다.
- 포탄 충돌은 현재 위치 점검사뿐 아니라 이전 위치와 현재 위치 사이의 선분 거리로도 검사해 빠른 포탄이 작은 목표를 건너뛰는 문제를 줄인다.

## Current Infantry Anti-Tank Rules

- 일반 보병은 전차를 보면 기본적으로 엄폐를 찾는다.
- 같은 편 보병이 전차 주변에 3명 이상 있고, 너무 가까운 거리가 아니면 `harass-tank` 상태로 전차에 견제 사격을 한다.
- 보병 총격은 전차를 빠르게 파괴하지 않는다. 작은 피격 이펙트와 아주 낮은 칩 피해만 준다.
- AI 보병도 `classId`와 `equipmentAmmo`를 가진다. 현재 맵에는 양 팀마다 공병 2명이 있고 RPG 2발과 수리킷 2회를 가진다.
- 공병은 `rpg-attack` 상태에서 적 전차를 RPG로 공격한다. 너무 가까우면 피하고, 사선이 막혔거나 탄약이 없으면 일반 보병처럼 엄폐/견제로 돌아간다.
- 공병 RPG는 아군 보병, 승무원, 도보 플레이어가 폭발 반경 가까이에 있으면 발사하지 않는다.

## Current Engineer Support Rules

- 공병은 적 보병 접촉이 없고, 적 전차가 너무 가까운 위협이 아닐 때 손상된 아군 전차를 찾는다.
- 수리 대상은 살아 있고 같은 팀이며 HP가 약 94% 미만인 전차다. 플레이어 전차는 약간 더 우선해서 본다.
- 공병은 `repair-tank` 상태로 전차 뒤쪽/측면의 통과 가능한 위치를 잡고, 수리킷 사거리 안이면 수리킷 1회를 소모해 전차 HP를 회복한다.
- 수리 이동은 아직 복잡한 재경로 탐색이 아니라 직접 접근 가능한 전차를 우선한다. 먼 전차를 고치러 벽을 긁는 행동을 줄이기 위한 제한이다.

## Current Scout AI Rules

- 현재 맵에는 양 팀마다 정찰병 2명이 있다. `B-SCT-*`, `R-SCT-*`는 `scout` 병과와 `sniper` 무기를 가진다.
- `CommanderAI`는 정찰병에게 `recon` 명령을 따로 배정한다. 정찰병은 보병 분대와 같이 거점 중앙으로 들어가지 않고 `world.reconPoints`의 도시 외곽/도로 측면 감시 위치로 이동한다.
- `SquadAI.activeUnits()`는 정찰병을 제외해 분대 진형과 전차-보병 협동 진입이 정찰병 때문에 흔들리지 않게 한다.
- 정찰병은 정찰 지점에 도착하면 `recon-watch` 상태로 멈춰 감시한다. 적 보병을 먼 거리에서 보면 `recon-snipe` 상태로 정지 사격하고, 적이 너무 가까우면 `recon-evade`로 엄폐/후퇴한다.
- 정찰병은 일반 보병보다 넓은 시야로 적 전차와 보병을 보고한다.
- 보고된 적은 `game.teamReports`에 짧은 시간 유지된다.
- `CombatController`는 정찰 보고가 살아 있는 적을 직접 시야 밖에서도 교전 후보로 본다. 실제 발사는 기존처럼 포신 조준과 `hasClearShot()`이 통과해야 한다.

## Current Infantry Rendering Rules

- 보병 몸체와 무기는 렌더링에서 분리한다.
- 일반 보병/공병 무기는 몸 중앙선이 아니라 캐릭터 오른쪽 어깨 위치에 붙어 보이도록 그린다.
- 정찰병이 저격 자세일 때는 저격총이 중앙으로 들어오고, 포신 앞에 얇은 조준선을 표시한다.

## Current Bug Watchlist

- 보병-전차 몸체 겹침은 `physics.js`에서 막는다. 새 보병 이동 코드는 `tryMoveCircle(..., { blockTanks: true })`를 써야 한다.
- 일반 보병 총기는 전차에 아주 낮은 칩 피해만 준다. 실제 대전차 위협은 공병 RPG 같은 특수 장비로 분리한다.
- 병과별 AI는 공병 RPG v1, 공병 수리 v1, 정찰 보고 v1까지 들어갔다. 공병 후퇴/연막 재배치는 아직 미래 작업이다.
- AI 전차는 포구와 목표 사이 사선에 아군 보병, 승무원, 도보 플레이어가 있으면 발사를 보류한다.
- 보병 AI는 아군 전차 포신 앞 사선에 걸리면 짧게 측면으로 빠지는 `avoid-fire-lane` 상태를 사용할 수 있다.

## Recommended Next Order

1. 확장된 맵 기준으로 거점 위치, 엄폐물 밀도, 기지 게이트 폭을 실제 플레이 후 한 번 더 조정한다.
2. 정찰 보고와 사선 위험을 HUD/디버그 오버레이에서 더 명확히 보이게 한다.
3. 공병이 후퇴 중 연막 뒤로 재배치하거나 수리킷을 재보급받는 행동을 추가한다.
4. AP/HE/RPG/수리킷 수치를 실제 플레이 기준으로 한 번 더 조정한다.
