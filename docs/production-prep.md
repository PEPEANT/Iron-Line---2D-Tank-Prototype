# Production Prep

현재 결론: 이 프로젝트는 정적 HTML/JS 게임으로 유지하고, 제작 전 기본 준비는 로컬 서버 실행, 문법 검사, 데스크톱/모바일 가로 화면 로딩 검수로 잡는다.

## 실행 기준

- `npm start`: 의존성 없는 Node 정적 서버를 `http://127.0.0.1:4173/index.html`에 띄운다.
- `npm run check`: `src/` 아래 JavaScript 파일을 `node --check`로 검사한다.
- `editor.html`: 같은 서버에서 맵 편집기를 연다.

## 현재 검수 상태

- Node `v24.13.1`에서 `node --check` 문법 검사를 통과했다.
- 데스크톱 `1365 x 768`에서 배치 화면 로딩과 초기 콘솔 에러 없음까지 확인했다.
- 모바일 가로 `844 x 390`에서 배치 화면이 화면 안에 들어오고 초기 콘솔 에러 없음까지 확인했다.
- Playwright 환경에서는 백그라운드/저빈도 프레임 때문에 5초 카운트다운이 실제 시간보다 느리게 흐를 수 있다. 실제 플레이 검수는 활성 브라우저 탭에서 확인한다.

## 제작 전 체크리스트

- 실제 플레이로 거점 위치, 엄폐물 밀도, 기지 게이트 폭을 다시 본다.
- 출격 후 전차/보병 AI가 3분 이상 중단 없이 교전하는지 본다.
- 모바일 가로에서 조이스틱, 발사, 탑승, 탄약 버튼이 손가락 위치와 겹치지 않는지 본다.
- 맵 편집기에서 저장/복사/다운로드 동선이 막히지 않는지 본다.
- `.codex-push/repo`는 배포용 복사본처럼 보이므로, 최종 배포 전 루트 파일과 차이가 남아 있는지 확인한다.
